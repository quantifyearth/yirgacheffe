# Contribution

Yirgacheffe is an open source project and aims to support a wide range of use cases, and thus welcomes a wide range of inputs: be they usage based (this is broken.../this isn't explained well in the docs.../we could support this project if we just had this feature...), technically lead (there's a better/more efficient way of doing this...), etc.

# Release process

## Version numbers

When an update is about to be pushed to pypi, a new version number is needed. This is the rough algorithm used for the version number changes:

* Major version number: API breaking changes, Major new feature areas
* Minor version number: New APIs added, significant bug fixes
* Point release number: Minor bug fixes and documentation improvements

There is always going to be awkward updates that break this, but this is the rough guide the project follows.
